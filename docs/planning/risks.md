# Riscos — Corretor IA

## Riscos técnicos

| Risco                                                                 | Impacto                                                       | Probabilidade                             | Mitigação                                                                                                                                        |
| --------------------------------------------------------------------- | ------------------------------------------------------------- | ----------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| Vazamento de dados entre corretores por falha de isolamento (IDOR)    | Crítico — quebra de confiança e possível violação de LGPD     | Média                                     | Isolamento aplicado centralmente em `server/policies`; teste de integração obrigatório em toda fase que toca dado por corretor (RN-026)          |
| Custo de infraestrutura de IA acima do esperado                       | Alto — inviabiliza operação de baixo custo (RNF-049)          | Média                                     | Limite de geração por plano (RN-070) desde o MVP; timeout e cache de falhas; monitoramento de consumo                                            |
| Qualidade inconsistente do conteúdo gerado por IA (invenção de dados) | Alto — dano à credibilidade do corretor perante seus clientes | Média-alta                                | Prompt restritivo com dados estruturados apenas do imóvel; testes automatizados verificando ausência de dados inventados em cenários controlados |
| Performance degradada em catálogos com muitos imóveis/fotos           | Médio-alto — abandono de visitantes no celular                | Média                                     | Paginação/carregamento progressivo desde a Fase 5; testes de carga com 20/100/500 imóveis (RNF-025)                                              |
| Upload malicioso disfarçado de imagem                                 | Crítico — comprometimento de storage/servidor                 | Baixa-média                               | Validação de MIME real, rejeição de executáveis, nomes aleatórios (RN-035, RN-036)                                                               |
| Dependência de biblioteca de autenticação com mudança de API          | Médio                                                         | Baixa                                     | ADR-0002 revisitado a cada mudança relevante de versão; testes de integração cobrindo o fluxo completo                                           |
| Acoplamento acidental a um único provedor de IA                       | Médio — dificulta troca futura                                | Média se não houver disciplina de revisão | Interface `AiContentProvider` obrigatória (ADR-0004); revisão de código verifica ausência de import direto de SDK de IA fora de `lib/ai`         |
| Migração de banco de dados mal planejada causando perda de dados      | Crítico                                                       | Baixa                                     | Migrações sempre versionadas e revisadas antes de aplicar; backups antes de mudanças estruturais (Fase 1 em diante)                              |
| Testes E2E frágeis (flaky) reduzindo confiança no pipeline            | Médio                                                         | Média                                     | Uso de localizadores semânticos e asserções de estado em vez de esperas fixas (ver `docs/quality/test-strategy.md`)                              |

## Riscos de produto

| Risco                                                                                                        | Impacto                                   | Probabilidade                | Mitigação                                                                                                                                                |
| ------------------------------------------------------------------------------------------------------------ | ----------------------------------------- | ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Corretores não considerarem a proposta de valor suficiente para pagar                                        | Alto — invalida a hipótese central do MVP | Média                        | Priorizar simplicidade extrema no onboarding e no cadastro; medir métricas de ativação definidas em `docs/product/product-vision.md`                     |
| Complexidade do cadastro de imóvel afastar usuários com baixa familiaridade tecnológica                      | Alto                                      | Média-alta                   | Fluxo em etapas curtas, salvamento automático de rascunho, uploads simples pela câmera                                                                   |
| Corretores desconfiarem de conteúdo gerado por IA                                                            | Médio                                     | Média                        | Sinalização transparente de conteúdo gerado por IA; edição sempre possível antes de publicar                                                             |
| Expectativa de recursos fora do escopo do MVP (CRM, integrações com portais) gerar frustração                | Médio                                     | Média-alta                   | Comunicação clara do escopo do MVP (`docs/product/mvp-scope.md`) como validação, não produto final                                                       |
| Concorrência de ferramentas já estabelecidas (portais imobiliários, Canva, grupos de WhatsApp)               | Médio                                     | Alta                         | Foco na integração do fluxo completo (cadastro → catálogo → anúncio → arte → compartilhamento) como diferencial, não em recursos isolados                |
| Falta de validação jurídica dos documentos legais (Termos de Uso, Política de Privacidade) antes de uso real | Alto — risco legal/regulatório            | Média se não tratado a tempo | Documentos marcados explicitamente como modelos iniciais; revisão jurídica obrigatória antes da operação comercial (ver `docs/security/threat-model.md`) |

## Acompanhamento

Estes riscos devem ser revisitados ao final de cada fase (especialmente
na Fase 10 — Hardening) e atualizados conforme evidência real de uso ou
mudança de contexto do projeto.
